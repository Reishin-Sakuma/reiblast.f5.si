# 24歳から見た「Claudeの世界」とは

<u>「安全なAIを作る、と言い続けた会社のAIを、結局一番信頼して使ってる。」</u>

どうも、レイと申します。

今日3本目の記事です。さすがにバテてきた。笑

ChatGPTの記事で「今はClaudeをメインで使ってます、理由は次の記事で」と書いて、Copilotの記事でも「しばらくClaudeでいいかな」と書いてたので、いい加減ちゃんと書きます。

有言実行（少し遅れたけど）。

---

さて今回は「**[Claude（クロード）](https://en.wikipedia.org/wiki/Claude_(language_model))**」について書きます。

知名度的にはChatGPTに劣りますが、使い込んでいくうちに「あ、これが自分には合う」となったAIです。

なぜそう思ったのかを、インフラエンジニア目線で語ります。

### そもそもClaudeって？

「ChatGPT知らない人いないでしょ」って思う方もいるかもしれないですけど、このブログの通例なんで書きます。（何回目だよって感じですが、これでたぶん最後です。たぶん。）

Claudeとは、**[Anthropic](https://ja.wikipedia.org/wiki/Anthropic)**というアメリカのAI企業が開発した「**[大規模言語モデル（LLM）](https://ja.wikipedia.org/wiki/%E5%A4%A7%E8%A6%8F%E6%A8%A1%E8%A8%80%E8%AA%9E%E3%83%A2%E3%83%87%E3%83%AB)**」です。

<span style="color:#8f20ff">「また別のAI？どれも同じじゃないの？」</span>

そう思う気持ち、わかります。ChatGPTもCopilotもあって、さらにClaude？って感じですよね。

でも、Claudeには他のAIとは少し違う「出自」があります。

### Anthropicって何者？

Claudeを作った<strong>[Anthropic](https://ja.wikipedia.org/wiki/Anthropic)</strong>という会社、実は<strong>[OpenAI](https://ja.wikipedia.org/wiki/OpenAI)の元スタッフが立ち上げた会社</strong>なんです。

「え、ライバルじゃん」と思うかもしれないですが、これがけっこう重要な話で。Anthropicの設立者たちは、「AIを安全に開発することにもっと真剣に取り組みたい」という思想を持ってOpenAIを離れたとされています。

そのあたりの経緯に詳しい人は調べてみてください。AIの覇権争いのドラマが見えてきて面白いです。笑

で、Anthropicが掲げているのが「**[Constitutional AI（憲法的AI）](https://en.wikipedia.org/wiki/Constitutional_AI)**」という考え方です。

簡単に言うと、<mark style="background-color:#ffcc00">**「AIがどう振る舞うべきかをルールセット（＝憲法）として明示的に定義して、AIが自分でその原則に従って判断できるように設計する」**</mark>アプローチです。

「有害なことは言わない」「嘘をつかない」「不確かなことは不確かと言う」みたいな原則をAIに叩き込む、という感じ。

<span style="color:#8f20ff">「結局それって口だけなんじゃないの？」</span>

正直最初はそう思ってました。笑

でも使ってみたら「あ、なんか確かに違う」と感じる場面が多かった。

## Claudeの「強み」と「弱み」

### 強み

**<span style="color:#0000ff">「わからない」をちゃんと言ってくれる</span>**

これが一番大きい。ChatGPTって、知らないことでも自信満々に答えてきたりするじゃないですか。いわゆる「[ハルシネーション](https://ja.wikipedia.org/wiki/%E3%83%8F%E3%83%AB%E3%82%B7%E3%83%8D%E3%83%BC%E3%82%B7%E3%83%A7%E3%83%B3_(%E4%BA%BA%E5%B7%A5%E7%9F%A5%E8%83%BD))」というやつです。

Claudeも嘘をつくことはあります。でも、「それは確かめられていません」「最新情報は確認してください」みたいな**留保を示してくれる頻度が高い**と感じてます。

「知らないことを知らないと言える」のは、実際の仕事で使う上でかなり重要で。

**<span style="color:#0000ff">文脈を長く保持できる（コンテキストウィンドウが広い）</span>**

ChatGPTも最近は伸びてきてますが、Claudeは昔から**長い文章を一度に処理するのが得意**でした。

長いドキュメントやログファイルをそのまま貼り付けて「ここの異常を見つけて」とか、「このコードを全部読んで、設計の問題点を教えて」みたいな使い方ができる。

インフラエンジニアってログが長いんですよ。笑

**<span style="color:#0000ff">文章の品質がいい</span>**

これは主観もありますが、生成される文章の自然さ・丁寧さがChatGPTより一段上な気がしています。

技術文書の下書きや、説明文の生成で特にそれを感じます。

**<span style="color:#0000ff">Claude Code（ターミナルで使えるやつ）が便利</span>**

これが最近ハマってる機能で、ターミナルからClaudeを呼び出してコードを書いてもらったり、ファイルの解析をしてもらったりできます。

エディタに依存しないので、サーバ作業中でもそのままCLIで使えるのがインフラエンジニア的にはポイント高い。

### 弱み

**<span style="color:#ff0000">知名度・普及度がChatGPTに負けている</span>**

周りのエンジニアに「Claude使ってる？」と聞くと、「え、何それ」ってなることがまだ多い。外部の人と話すとき「Claudeで調べて」が通じないのが地味につらい。笑

**<span style="color:#ff0000">サードパーティ連携が少ない</span>**

ChatGPTやCopilotはいろんなサービスと連携できますが、Claudeはまだ選択肢が少ない。

まあ個人利用の範囲では困らないですが。

**<span style="color:#ff0000">画像を「生成」はできない</span>**

ClaudeはLLM（テキスト系）に特化していて、画像を作り出すことはできません。

画像を「読む」（解析する）はできるので、スクリーンショットを見てもらって「このエラー何？」みたいな使い方はできます。あくまで「生成」ができないということ。

**<span style="color:#ff0000">料金プランが若干高め</span>**

個人プランの Claude Pro は月$20（約3,000円）。ChatGPT Plusと同じ料金感ですが、無料枠の制限がやや厳しい印象。

ただ、使い込むなら払う価値は十分あると思ってます。

## こんなときに使えるよ（ユースケース）

インフラエンジニアとして実際に「助かった」と思ったケースです。

- **長いログの解析** - エラーログをそのまま貼り付けて「この中で異常なパターンは？」と聞く。数百行でも読んでくれる
- **技術文書の下書き** - 設計書や手順書の初稿を出してもらって、自分で肉付け。一から書くより断然速い
- **構成レビューの壁打ち** - 「この構成で懸念点は？」「代替案を出して」みたいな設計相談
- **スクリプトのデバッグ** - エラーとコードを貼り付けて「なんでこうなる？」と聞く。コードが長くても読んでくれる
- **英語技術文書の翻訳・要約** - 公式ドキュメントが英語でつらいとき。精度が高くて助かってる

逆に<u>向いてないケース</u>。

- **最新情報が必要なとき** - 学習データのカットオフがあるので、最近リリースされた機能とかは知らないことがある。検索エンジンや[Perplexity](https://ja.wikipedia.org/wiki/Perplexity_AI)を使ったほうがいい
- **画像生成** - Claudeではできないので[DALL·E](https://ja.wikipedia.org/wiki/DALL-E)や[Stable Diffusion](https://ja.wikipedia.org/wiki/Stable_Diffusion)を使う
- **社内固有の知識** - 当然Claudeも知らないので、社内Wikiに頼る

基本的には、<strong>「調査・分析・下書き」に使える万能型</strong>という印象です。

## ChatGPTからClaudeに乗り換えた理由、正直に言います

これが今回の記事の本題です。

最初はみんなと同じようにChatGPTを使っていたんですが、ある時から「なんかClaudeのほうが合うな」と感じるようになって。

理由を振り返ってみると、こういう感じです。

**答えの「誠実さ」が違う**

ChatGPTは答えてくれるのはいいんですが、「それってほんとに正しいの？」と思いながら使い続けないといけない緊張感がある。

Claudeは「わからない場合はわからない」「不確かな情報は不確かと前置きする」というスタンスが多くて、<span style="color:orange">**「こいつ、嘘をつきにくそうだな」**</span>という信頼感が生まれたんですよね。

**長い作業に付き合ってくれる**

インフラの設計レビューとか、長いログ解析とかって、どうしても長い会話になります。

ChatGPTは会話が長くなると途中で前の文脈が薄れる感じがしましたが、Claudeはその点で安定してる気がする。

**文章のクオリティが好み**

技術文書の下書きを作ってもらったとき、Claudeのほうが自然で読みやすい文章を出してくれた。主観ではありますが、これがけっこう決め手になりました。

---

ちなみに正直に言うと、このポートフォリオサイト、Claudeとめっちゃ会話しながら作りました。コード書いてもらったり、構成を相談したり。

<span style="color:#8f20ff">「Claudeを使って、Claudeのブログを書いてる」というメタい状況になってます。笑</span>

## じゃあエンジニアとしてどう向き合う？

今日3記事書いてきて改めて思うのは、どのAIも「**ツールとして使う**」という姿勢が大事だということ。

Claudeがいくら「誠実」でも、最終的に判断するのは自分です。AIの言うことを鵜呑みにして本番環境に適用したらえらいことになる、という話は変わらない。

<span style="color:orange">**「AIは優秀なアシスタント、でも決定権は自分にある」**</span>

これを常に頭に置いておく、というのが今の自分のスタンスです。

「Claudeが言ってたんで」は仕事の言い訳にならないですからね。笑

### こんなもんかな？

今日はChatGPT・GitHub Copilot・Claudeと3記事書きました。さすがに疲れた(^o^)

それぞれに得意不得意があって、使い分けが大事だなと改めて思いました。

ざっくりまとめると。

- <span style="color:#0000ff">**ChatGPT**</span> → 生成AIへの入口。なんでも聞ける物知りな友達
- <span style="color:#0000ff">**GitHub Copilot**</span> → エディタに常駐する、コード特化のアシスタント
- <span style="color:#0000ff">**Claude**</span> → 長い作業・文書系・相談ごとで頼れる、今の自分のメイン

次は「[Perplexity](https://ja.wikipedia.org/wiki/Perplexity_AI)」について書こうと思ってたんですが、今日は体力的に無理なので一旦おわりにします。笑

あと最近は Claude Code を本格的に使い始めていて、これがまたいい感じです。別途記事にするかもしれない。

それでは！
