# 24歳から見た「Claudeの世界」とは

<u>「嘘をつかないように設計されているAIに、なぜか人間味を感じた。」</u>

どうも、レイと申します。

昨日、勢いで3本記事書いてバテました。笑

今日は祝日で休みなので、昼間からぼんやりしてたんですが、夕方になって「ああ、Claudeの話をちゃんとしてないな」と思ったので書きます。

沖縄、昨日も今日も日中は暖かかったです。2月の祝日に半袖で過ごしてました。本土の人には絶対信じてもらえないやつ。

---

昨日の記事を読んでくれた方はわかると思うんですが、ChatGPTの記事でも、GitHub Copilotの記事でも、締めのところに「今はClaudeをメインで使ってます」「しばらくClaudeでいいかな」ってさらっと書いて終わってるんですよね。

2回も名前を出しておいてちゃんと語らないのはフェアじゃないな、と。

ということで今日は「**なぜChatGPTじゃなくてClaudeなのか**」を、ツールレビューじゃなく個人的な惚れた話として書きます。

## きっかけは、ChatGPTに「もう無理」と言われたこと

最初にClaudeを使うきっかけになったのは、仕事でけっこう長いログを ChatGPT に貼り付けようとしたときです。

サーバの障害対応で、複数サービスのエラーログをまとめて「相関性を見つけてほしい」と投げようとしたら、入力上限に引っかかって受け付けてもらえなかった。

<span style="color:#8f20ff">「え、これ入らないの？」</span>

その場しのぎで「Claude、コンテキスト長いって聞いたことあるな」と思って試してみたのが最初です。

動機が不純www

で、実際に試してみたらするっと受け付けてくれて、しかも解析の回答もなかなかいい。

「なんだ、普通に使えるじゃん」というのが正直な第一印象でした。

## 「あれ、ChatGPTと何か違う」と思い始めたこと

そこからしばらく並行して使い続けていたんですが、少しずつ「なんか違うぞ」と感じる場面が増えていきました。

一番最初に気づいたのは、**Claudeが「わからない」と言ったとき**です。

あるサービスの最新バージョンの設定方法を聞いたとき、Claudeが「私の学習データが古いため、最新版については確認していただく必要があります」ってはっきり言ってきた。

同じことをChatGPTに聞いたときは、それっぽい回答は来たんですが、あとで確認したらそのバージョンの設定方法ではなかった。

<span style="color:#ff0000">**ChatGPTは嘘をつくとき、自信満々につく。**</span>

いや、Claude も嘘はつくんです。でも頻度と「誠実さ」が違う。「不確かなことは不確かと言う」というのがちゃんと体感できて、それ以来 Claude を信用するようになりました。

---

この「誠実さ」には、Claudeを作った<strong>[Anthropic](https://ja.wikipedia.org/wiki/Anthropic)</strong>という会社の思想が関係しています。

Anthropicは<strong>[OpenAI](https://ja.wikipedia.org/wiki/OpenAI)</strong>の元スタッフが「AIを安全に開発することにもっと真剣に向き合いたい」という考えから設立した会社で、<strong>「[Constitutional AI](https://en.wikipedia.org/wiki/Constitutional_AI)（憲法的AI）」</strong>というアプローチを掲げています。

要するに、「AIがどう振る舞うべきかをルールとして明示的に定義して、そのルールに基づいて自分で判断できるように設計する」というものです。

最初聞いたとき「なんか理念先行っぽくない？」と思ってたんですが、実際に使ってみると確かに違う気がする、というのが今の正直な感想です。

## 使い始めて9ヶ月、気づいたらClaudeなしで仕事できなくなってた

最初のきっかけはさっき書いた通りで、ChatGPTにログをはじかれたからです。でもそこから9ヶ月くらい使い続けていくうちに、気づいたら「まずClaudeに聞く」という習慣になっていました。

設計の壁打ち、手順書の下書き、長いログの解析、英語ドキュメントの要約。仕事のいたるところでClaudeが入り込んでいた。

で、「ああ、完全に依存してるな」と自覚したのが、このポートフォリオサイトを作ったときです。

使ったのは**Claude Code**というCLIのツールで、ターミナルから直接Claudeを呼び出してコードの相談やファイルの解析ができるやつです。[Astro](https://ja.wikipedia.org/wiki/Astro_(%E3%83%95%E3%83%AC%E3%83%BC%E3%83%A0%E3%83%AF%E3%83%BC%E3%82%AF))を使った静的サイト構築はほぼ初めてで、コンポーネントの設計から画像の扱い方まで、ターミナルを離れずにそのまま相談しながら進められた。

エディタに縛られないので、SSHでつないだサーバ作業中でもそのまま使えるのがインフラ寄りの人間には刺さります。

<span style="color:#8f20ff">「Claude Code を使って、Claude の記事を書いてる」</span>というメタい状況ではあるんですが、それくらいには信頼してます。笑

## 不満がないわけじゃない

推し語りになってきたので、フェアに言うとダメなところも書きます。

一番困るのは**知名度の低さ**です。

職場で「Claude使ってる？」って聞くと「何それ？」ってなることが多い。外の人と生成AI の話をしても「ChatGPTは知ってる」で会話が完結して、Claudeを布教できる機会がなかなかない。

あとは、**無料で使える範囲が狭い**。

Claude Pro（有料プラン）は月$20で、頻繁に使うなら課金は必要になってきます。ChatGPT の無料版でとりあえず試したい、みたいな入り口のしやすさでは負けてます。

画像の「生成」もできないので（「読む」はできる）、そこに期待してる人は<strong>[DALL·E](https://ja.wikipedia.org/wiki/DALL-E)</strong>や<strong>[Stable Diffusion](https://ja.wikipedia.org/wiki/Stable_Diffusion)</strong>を別に使う必要がある。

ただ、テキスト生成・長文処理・相談相手、という自分の使い方にはこれで十分なので、今のところ不満より満足のほうがデカいです。

## 結局、何が言いたかったか

昨日3記事書いた流れで言うと。

- <span style="color:#0000ff">**ChatGPT**</span> は生成AIの入口として最高。なんでも相談できる物知りな友達
- <span style="color:#0000ff">**GitHub Copilot**</span> はエディタに常駐するコード補完特化のアシスタント
- <span style="color:#0000ff">**Claude**</span> は長い作業・文書まわり・深い相談に向いてる、今の自分のメイン

というポジションです。

どれが一番偉いとかではなく、それぞれに向き不向きがあって、目的に応じて使い分けるのが正解だと思ってます。

<span style="color:orange">**ただ一つだけ選べと言われたら、今の自分はClaudeです。**</span>

以上が、2回も「Claudeメインです」と書きながら語ってこなかったやつの答えです。笑
