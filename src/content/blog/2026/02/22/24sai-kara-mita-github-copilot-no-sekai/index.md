# 24歳から見た「GitHub Copilotの世界」とは

<u>「コードを書く速度より、AIをうまく使う速度のほうが大事になってきた。」</u>

どうも、レイと申します。

さっきChatGPTの記事を書いたばかりなんですが、勢いそのままに続きを書きます。笑

「次はClaude・Copilotについて書く」って締めたので、有言実行ということで。

---

さて今回は「**[GitHub Copilot](https://ja.wikipedia.org/wiki/GitHub_Copilot)**」について書きたいと思います。

ChatGPTは「なんでも答えてくれるAI」って感じですが、Copilotは少し毛色が違います。

どう違うのか、インフラエンジニア目線で語ります。

### そもそもGitHub Copilotって？

「ChatGPT知らない人いないでしょ」って思う方もいるかもしれないですけど、このブログの通例なんで書きます。（20歳の仕事の記事で「そこから？」って言われたやつの三回目です。懲りない。）

GitHub Copilotとは、**[GitHub](https://ja.wikipedia.org/wiki/GitHub)**（[Microsoft](https://ja.wikipedia.org/wiki/Microsoft)傘下）が提供する「**AIコードアシスタント**」です。

[Visual Studio Code](https://ja.wikipedia.org/wiki/Visual_Studio_Code)などのエディタに拡張機能として入れることで、コードを書いているときにリアルタイムで「次に書くべきコード」を提案してくれます。

ChatGPTは「チャットで質問して答えをもらう」スタイルですが、Copilotは「**エディタの中で、書きながらAIが横に立って一緒に考えてくれる**」スタイルです。

この違いが結構大事で、使い勝手がだいぶ違うんですよね。

### 「コード補完AI」ってなんやねん

GitHub Copilotは「**AIコード補完**」ツールのひとつです。

「コード補完」自体は昔からある機能で、例えばエディタで `pr` と打ったら `print` って候補が出てくる、あれです。

Copilotはそれの超進化版で、コメントで「// サーバのホスト名を取得する関数」と書くだけで、その下に関数を丸ごと生成してくれたりします。

<span style="color:#8f20ff">「それChatGPTでよくない？」</span>

そう思いますよね。実際僕も最初そう思ってた。

違いはここで、Copilotは**今開いているファイルのコードの文脈を読んで**提案してくれます。

すでに書いてある関数の命名ルールや、使っているライブラリの書き方に合わせた提案が出てくるので、ChatGPTより「馴染む」んです。

（興味ある人は「[OpenAI Codex](https://ja.wikipedia.org/wiki/OpenAI_Codex)」あたりで調べてみてください。Copilotの土台になったモデルです）

## GitHub Copilotの「強み」と「弱み」

### 強み

**<span style="color:#0000ff">エディタを離れずに補完が受けられる</span>**

これが一番でかい。ChatGPTだとブラウザに切り替えてプロンプト書いてコピペして…という手間がありますが、Copilotはエディタの中で完結します。Tab キー一発で提案を受け入れられる快感、一度味わったら戻れない。

**<span style="color:#0000ff">コードの文脈を理解して提案してくれる</span>**

先述した通り、プロジェクト内の他ファイルや、同じファイル内のコードを参考にした提案が出てきます。「このプロジェクトっぽい書き方」になるので、ChatGPTより手直しが少なくて済む場面が多い。

**<span style="color:#0000ff">コメント→コードの変換が便利</span>**

「// 〇〇する処理」とコメントを書くだけでコードを生成してくれるのが地味にありがたい。設計図→実装のギャップを埋めてくれる感じ。

### 弱み

**<span style="color:#ff0000">有料</span>**

個人プランは月$10（約1,500円）。ChatGPTの無料版に慣れると、「あれ、お金いるの？」って壁に当たります。

ただし**学生や認定オープンソース開発者は無料**で使えます。

**<span style="color:#ff0000">ChatGPTより説明が苦手</span>**

Copilotはあくまでコード補完に特化したツールなので、「このコードの設計はどう思う？」とか「このアーキテクチャの問題点は？」みたいな相談はChatGPTのほうが向いてます。

用途が違うので、どっちが上ということではないんですけどね。

**<span style="color:#ff0000">提案が合わないこともある</span>**

文脈を読んでくれるとはいえ、的外れな提案が来ることも普通にあります。特に、プロジェクト固有の複雑なロジックや、あまり一般的でないライブラリを使っているときは精度が落ちる印象。

あとはChatGPT同様、**提案が正しいかどうかは自分で判断する必要**があります。

## こんなときに使えるよ（ユースケース）

インフラエンジニアって、そんなにコード書かないのでは？って思う人もいるかもですが、最近はシェルスクリプトとか[PowerShell](https://ja.wikipedia.org/wiki/PowerShell)、[Python](https://ja.wikipedia.org/wiki/Python)、[Ansible](https://ja.wikipedia.org/wiki/Ansible)あたりは普通に書きます。

そういう場面で役に立ったケースを挙げます。

- **シェルスクリプトの補完** - ループ処理とかエラーハンドリングの定番パターンをサっと出してくれる
- **PowerShell補完** - サーバ操作系のコマンドレット名って覚えづらいんですが、補完で候補を出してくれるのが助かる
- **Ansibleのplaybook作成** - `- name: ` と書いただけで次のtaskの雛形を出してきてくれる。感動した
- **コメントの補完** - ドキュメントコメントも書いてくれるので、引き継ぎ資料作成のときに地味に便利
- **同じような処理の繰り返し** - 似たパターンのコードを複数書くとき、2つ目以降はほぼ自動で出てくる

逆に<u>向いてないケース</u>。

- **設計の相談** - Copilot Chatでもある程度はできるけど、深い相談はChatGPTのほうが向いてる
- **調査・情報収集** - コード補完ツールなので、「〇〇って何？」みたいな質問はChatGPTに聞いたほうが早い
- **機密コードの補完** - 入力したコードがGitHubに送信される可能性があるので、業務利用の際はセキュリティポリシーを確認すること

要するに、<strong>ChatGPTは「相談相手」、Copilotは「隣で一緒に手を動かしてくれる人」</strong>です。

## インフラエンジニア、GitHub Copilot使ってみた

正直に言うと、最初は「コードあんまり書かないし、自分には必要ないかな」と思ってました。

でも使ってみたら、スクリプト書く時間が体感で半分くらいになった気がします。

一番驚いたのが、コメントで意図を書くだけでコードを出してくれること。

例えば、`# ホスト名のリストをCSVから読み込んで、pingが通るか確認する` と書いたらそれっぽいコードが出てきた。精度はともかく、「たたき台」として十分すぎた。

<span style="color:#ff0000">**「え、これもう自分でコード書かなくてよくない？」**</span>

と思いましたが、やっぱりそうはならない。

出てきたコードが「意図通りの動きをするか」「セキュリティ上の問題がないか」「既存の環境に合っているか」を判断するのは自分でやらないといけないので。

ここはChatGPTのときと同じで、**知識がある人が使うからこそ活きる**ツールだなと実感しています。

## まとめ

ChatGPTとGitHub Copilotは、生成AIという共通点はあるけど、**使い方が全然違います**。

- <span style="color:#0000ff">**ChatGPT**</span> → なんでも相談できる物知りな友達
- <span style="color:#0000ff">**GitHub Copilot**</span> → 隣に座って一緒にコード書いてくれる相棒

どちらか一方でいい、じゃなくて**両方を目的に合わせて使い分けるのが正解**だと思います。

<span style="color:orange">**「とりあえず両方触ってみること」**</span>が大事なので、ChatGPTでAIに慣れてきたらCopilotも試してみてください。
