# 24歳から見た「Perplexityの世界」とは

<u>「ChatGPTが嘘をつき、Claudeが古いなら、調べ方ごと変えてしまえばいい。」</u>

どうも、レイと申します。

昨日のClaudeの記事の締めで「次はPerplexityかな」ってさらっと書いたんですが、有言実行します。

沖縄、今日は少し雲が多かったけど気温は相変わらず高め。夕方に外に出たら半袖で余裕でした。2月の沖縄、本当にチートすぎる。本土の友達に言ったら「うらやましい」って毎回言われるんだけど、沖縄の夏の湿度と引き換えだからね（^o^）

---

さて今回は「<strong>[Perplexity](https://ja.wikipedia.org/wiki/Perplexity_AI)</strong>」について書きます。

ChatGPT・GitHub Copilot・Claude と書いてきましたが、今回は**毛色がだいぶ違います**。

前の3つは「**生成AI**」、つまり「文章や回答を新しく作り出すAI」でした。

Perplexity は見た目や使い方は似てるんですが、本質が違う。

一言で言うと、<mark style="background-color:#ffcc00"><strong>「AIを使ってリアルタイムにウェブを検索し、出典付きで答えを返してくれるサービス」</strong></mark>です。

### そもそもPerplexityって？

「このブログの通例なんで書きます。」（もはやこのくだりが様式美になってきてる笑）

<strong>[Perplexity AI](https://ja.wikipedia.org/wiki/Perplexity_AI)</strong>とは、2022年にアメリカで創業されたスタートアップが提供するAIサービスです。

Googleのような検索エンジンに見えて、ChatGPTのような会話型AIでもある、という不思議なポジションにいます。

この両方の特徴を掛け合わせた「**AIアンサーエンジン**」と呼ばれたりします。

普通の検索エンジンと何が違うかというと、「関連しそうなサイトの一覧を並べる」のではなく、**ウェブを実際に検索した上で、その内容をまとめた回答を出してくれる**んです。

しかも、ちゃんと<span style="color:#0000ff">**出典（ソース）を明示してくれる**</span>。「このURLを参照した」がセットで出てくるので、「本当？」と思ったら自分で確認できる。

ChatGPTのときに書いた「[ハルシネーション](https://ja.wikipedia.org/wiki/%E3%83%8F%E3%83%AB%E3%82%B7%E3%83%8D%E3%83%BC%E3%82%B7%E3%83%A7%E3%83%B3_(%E4%BA%BA%E5%B7%A5%E7%9F%A5%E8%83%BD))」を覚えてますか？<span style="color:#8f20ff">「知らないことでもそれっぽく答える問題」</span>

あれが起きにくいのが Perplexity の大きな特徴です。なぜかというと、**Perplexityはウェブをリアルタイムで検索して、実在するページの情報をもとに答えるから**。「でっち上げる」のではなく「引用してまとめる」に近い。

### 「それ、Googleでよくない？」

絶対思いますよね。笑 僕も最初そう思ってました。

確かに「調べ物はとりあえずGoogleで検索」でずっとやってきた。でも使い比べてみると、違いがわかります。

[Google](https://ja.wikipedia.org/wiki/Google_%E6%A4%9C%E7%B4%A2) で「〇〇とは」と検索すると、「関連するWebページが数十〜数百件」出てきます。

Perplexity で同じことを聞くと、「ポイントをまとめた回答 + 出典3〜5件」が返ってきます。

**答えを知りたいだけのとき、ググると何個もページを開いてスクロールして読んで比較して…という工程が発生する**んですよね。Perplexityはその工程をかなりショートカットしてくれます。

<span style="color:#8f20ff">「それって正確なの？」</span>

というのが次の疑問だと思うんですが、出典が明示されるので、重要な情報は出典元をちゃんと確認するというフローになります。ググってもどうせ元のページは確認するわけなので、むしろ効率は上がってる感じ。

（仕組みに興味ある人は「<strong>[検索拡張生成（RAG）](https://en.wikipedia.org/wiki/Retrieval-augmented_generation)</strong>」あたりで調べてみてください。Perplexityが使ってる技術の近くにあります）

## Perplexityの「強み」と「弱み」

### 強み

**<span style="color:#0000ff">リアルタイムの情報が取れる</span>**

これが一番でかい。ChatGPTやClaudeには「知識のカットオフ」があって、最新情報には弱いんですよね。Perplexityはウェブをそのとき検索するので、**今日発表されたニュースだって答えてくれます**。

インフラ系だと、「このOSSの最新バージョンで変わったこと」「直近の脆弱性情報」「製品のEOLスケジュール」なんかを調べるのに助かってます。

**<span style="color:#0000ff">出典が明示される</span>**

「根拠は？」「どのページに書いてある？」がすぐわかる。ChatGPTが「自信満々に嘘をつく」問題と真逆の設計です。

誰かに確認してもらうときも「Perplexityにこう出てた、出典はここ」って言えるので、業務的にも使いやすいです。

**<span style="color:#0000ff">英語ドキュメントの要約が速い</span>**

インフラエンジニアあるあるだと思うんですが、MicrosoftやCiscoの公式ドキュメントって長いんですよね。あれをいちいち全部読むのがしんどい。

Perplexityに「〇〇の設定方法をまとめて」と投げると、必要な部分だけ引き出してくれます。英語の一次情報に強いのがありがたい。

**<span style="color:#0000ff">無料でも十分使える</span>**

無料プランでもそこそこ使えます。有料プランは月$20（Pro版）で、ChatGPTやClaudeと同等の価格帯。「調べ物専用」と割り切れば、無料で足りることが多いです。

### 弱み

**<span style="color:#ff0000">文章の「生成」は苦手</span>**

Perplexityは「調べて答える」のが得意であって、「文章を一から書く」のはChatGPTやClaudeのほうが向いてます。

報告書の下書き、メールの作成、コードのレビューは Perplexityより他のAIを選んだほうがいいです。用途が根本的に違うというか。

**<span style="color:#ff0000">情報の質が「検索結果次第」</span>**

ウェブの情報をもとに答えるということは、**ウェブに間違った情報があれば、その間違いを引用してしまう**リスクがある。

出典が明示される分、自分で確認はできるんですが、「Perplexityが言ってたから絶対正しい」は危ないです。<span style="color:#8f20ff">出典付き ≠ 正しい、というのは意識しておく必要があります。</span>

**<span style="color:#ff0000">深い思考・相談には向かない</span>**

「このシステム構成のリスクを整理して」とか「この判断、どう思う？」みたいな、**情報じゃなくて考えや判断を求める問い**はClaudeのほうが向いてます。

Perplexityは「探す」に特化してる感じで、「考える」壁打ち相手としてはやや物足りないです。

**<span style="color:#ff0000">ページ内の情報しか取れない</span>**

ウェブ上に公開されていない情報、例えば社内のドキュメントや有料データベースの内容は当然取れません。あくまで「ウェブ上の公開情報の整理」に限られます。

## インフラエンジニア、Perplexityと出会った話

正直に言うと、Perplexityを使い始める前は「ChatGPT・Claude・Googleがあれば十分じゃない？」と思ってたんですよ。

使い始めたきっかけも大して劇的じゃなくて、SNSで「Perplexityが便利」っていうポストを見て、ふーんって感じで試したくらいです。笑

でも使ってみたら、<span style="color:#ff0000"><strong>「あ、これ Googleより速いじゃん」</strong></span>となる場面が思ったよりあって。

特に助かったのは、業務中に「このソフトウェアのバージョン〇〇でのリリースノートを教えて」と調べたとき。

ググると公式のリリースノートページを探して、英語の長文を読んで、変更点をピックアップして…という工程が発生する。

Perplexityに聞いたら、主要な変更点を箇条書きでまとめてくれた上に、公式ドキュメントへのリンクが添付されて出てきた。

**調査して、整理して、出典付きで出してくれる**がワンストップで終わる体験が、最初は「え、こんなに楽なの」ってなりました。

---

今の僕の大まかな使い分けはこんな感じです。

| シーン | 使うツール |
|---|---|
| 最新情報・ニュース・脆弱性調査 | **Perplexity** |
| 事実確認・出典が必要な調査 | **Perplexity** |
| 文章の下書き・要約・深い相談 | **Claude** |
| コードのレビュー・生成 | **Claude / ChatGPT** |
| エディタ内のコード補完 | **GitHub Copilot** |

どれかひとつが全部できる、じゃないんですよね。**向き不向きがある**というのが、これだけ触ってきての率直な感想です。

## ChatGPT・Claude・Perplexity、何が違うの？

ここまで4つのAIを記事にしてきたので最後に整理します。

- <span style="color:#0000ff">**ChatGPT**</span> → なんでも相談できる物知りな友達。生成AIへの入口として最高。ただし嘘をつくことがある
- <span style="color:#0000ff">**GitHub Copilot**</span> → エディタに常駐して、コードを一緒に書いてくれる相棒
- <span style="color:#0000ff">**Claude**</span> → 長い文書・深い相談に向いてる、誠実なやつ。今の自分のメイン
- <span style="color:#0000ff">**Perplexity**</span> → リアルタイムで調べて、出典付きで教えてくれる情報収集特化型

この4つが揃ったことで、<u>「生成AI、使いこなせてる感」が一気に上がった気がしてます</u>。笑

それぞれに役割があって、適材適所で使い分けるのが正解。ひとつのツールに全部任せようとすると、どこかで限界が来ます。
