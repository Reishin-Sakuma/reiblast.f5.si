このブログでは `.mdx` 形式で記事を書くと、Markdownの記法に加えてリッチなコンポーネントが使えます。このデモ記事で実際の仕上がりを確認してみましょう。

## シンタックスハイライト

コードブロックに言語名を指定すると、自動でカラーハイライトされます。

### JavaScript / TypeScript

```typescript
interface BlogPost {
  title: string;
  pubDate: Date;
  tags: string[];
  draft: boolean;
}

async function getPublishedPosts(): Promise<BlogPost[]> {
  const posts = await getCollection('blog', ({ data }) => {
    return data.draft !== true;
  });

  return posts.sort((a, b) =>
    b.data.pubDate.valueOf() - a.data.pubDate.valueOf()
  );
}
```

### Python

```python
from datetime import datetime

def format_date(date: datetime) -> str:
    """日付を日本語フォーマットに変換する"""
    return date.strftime("%Y年%m月%d日 %H:%M")

posts = [
    {"title": "はじめての記事", "date": datetime(2026, 2, 10)},
    {"title": "2本目の記事",   "date": datetime(2026, 2, 8)},
]

for post in sorted(posts, key=lambda p: p["date"], reverse=True):
    print(f"{format_date(post['date'])} - {post['title']}")
```

### Bash

```bash
# 新しい記事を作成する

echo "記事を作成しました: src/content/blog/$YEAR/$MONTH/$DAY/$SLUG/index.md"
```

## Noteコンポーネント

4種類の情報ボックスが使えます。

<Note type="info">
  **情報 (info)** — 補足情報や関連知識を伝えるときに使います。記事の流れを邪魔せず、読者が気づいておくと良い情報を添えられます。
</Note>

<Note type="tip">
  **ヒント (tip)** — 便利なショートカットや効率化のコツなど、読者の作業を楽にする情報に使います。
</Note>

<Note type="warning" title="注意してください">
  **注意 (warning)** — 設定ミスやよくある落とし穴など、注意が必要な箇所に使います。タイトルはカスタマイズできます。
</Note>

<Note type="danger">
  **危険 (danger)** — データ削除・本番環境への影響など、取り返しのつかない操作の前に使います。
</Note>

## Detailsコンポーネント（折りたたみ）

長い補足説明や、上級者向けの情報を折りたたんで隠せます。

<Details summary="Process Monitor のインストール方法（クリックで展開）">
  1. [Sysinternals Suite](https://learn.microsoft.com/en-us/sysinternals/downloads/sysinternals-suite) からダウンロード
  2. ZIPを解凍して `Procmon64.exe` を取り出す
  3. **管理者権限**で起動する

  <Note type="warning">
    Process Monitor は起動中に大量のメモリを消費します。長時間の監視は避けてください。
  </Note>
</Details>

## テーブル

Markdownのテーブル記法もきれいに表示されます。

| コンポーネント | ファイル | 用途 |
|--------------|---------|------|
| `Note` | `Note.astro` | info / warning / tip / danger ボックス |

## まとめ

このブログでは `.mdx` 拡張子を使うことで、通常のMarkdown記法に加えて以下が使えます：

- **シンタックスハイライト** — 言語名を指定するだけ
- **目次（TOC）** — 見出しから自動生成（このページ右側を見てください）
- **Note** — 4種類の情報ボックス
- **Details** — 折りたたみセクション

新しい記事を書く際は `blog-template.md` をベースに、必要に応じて `.mdx` に切り替えてみてください。
